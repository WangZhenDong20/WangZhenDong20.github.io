设计模式之责任链模式

Posted by SleepingFace on Saturday, August 17, 2024

责任链模式（Chain of Responsibility Pattern）

概述：

责任链模式是一种行为型设计模式，它允许多个对象有机会处理请求，将这些对象连成一条链，直到有对象处理它为止。请求在链上传递，直到链上的某个对象处理该请求。

关键点：

  • 处理对象的动态组合：责任链模式将处理对象以链式结构组合，使得请求可以沿着这条链依次传递，直到有对象处理它。
  • 请求的传递：请求从链的开始传递到链尾，链中的每个对象都可以选择处理该请求或将请求传递给下一个处理对象。
  • 降低耦合：客户端无须知道请求是由哪一个对象处理的，从而降低了对象之间的耦合度。

举个例子：

现在我们需要有一个校验文件的需求，其中需要校验文件大小、文件格式、文件内容、文件中图片内容。

责任链模式就是将四个校验任务连接在一起形成一条链来执行。

我们可以把其中的每一个校验任务看做一个节点。

责任链模式的优点就是：

  • 可以将节点调换顺序
  • 可以新增节点
  • 也可以删除节点

因此，我们可以根据具体的需求来编排这个节点，从而达到目的。

主要解决的问题

  • 解耦请求发送者和接收者，使多个对象都有可能接收请求，而发送者不需要知道哪个对象会处理它。

使用场景

  • 当有多个对象可以处理请求，且具体由哪个对象处理由运行时决定时。
  • 当需要向多个对象中的一个提交请求，而不想明确指定接收者时。

责任链模式的结构

  1. Handler（处理者）：定义一个处理请求的接口，并且包含一个对下一个处理者的引用。
  2. ConcreteHandler（具体处理者）：实现Handler接口，处理它能够处理的请求，否则将请求传递给下一个处理者。
  3. Client（客户端）：创建责任链，并发出请求。

示例代码

示例场景：请假审批流程

假设在一个公司中，有一个请假审批流程。员工申请请假时，请假请求会按照职位级别传递，直到请求被批准或拒绝。一般流程可能是：员工的主管审批 -> 部门经理审批 -> 人力资源经理审批。

1.创建抽象处理者

// 抽象处理者
public abstract class LeaveHandler {
    protected LeaveHandler nextHandler;

    // 设置下一个处理者
    public void setNextHandler(LeaveHandler nextHandler) {
        this.nextHandler = nextHandler;
    }

    // 处理请求的抽象方法
    public abstract void handleRequest(int days);
}

2.创建具体处理者

// 具体处理者：主管
public class Supervisor extends LeaveHandler {
    @Override
    public void handleRequest(int days) {
        if (days <= 2) {
            System.out.println("Supervisor approved " + days + " days leave.");
        } else if (nextHandler != null) {
            nextHandler.handleRequest(days);
        }
    }
}

// 具体处理者：部门经理
public class DepartmentManager extends LeaveHandler {
    @Override
    public void handleRequest(int days) {
        if (days <= 5) {
            System.out.println("Department Manager approved " + days + " days leave.");
        } else if (nextHandler != null) {
            nextHandler.handleRequest(days);
        }
    }
}

// 具体处理者：人力资源经理
public class HRManager extends LeaveHandler {
    @Override
    public void handleRequest(int days) {
        if (days <= 10) {
            System.out.println("HR Manager approved " + days + " days leave.");
        } else {
            System.out.println("Leave request for " + days + " days was denied.");
        }
    }
}

3.创建客户端

// 客户端
public class LeaveRequestClient {
    public static void main(String[] args) {
        // 创建具体处理者
        LeaveHandler supervisor = new Supervisor();
        LeaveHandler departmentManager = new DepartmentManager();
        LeaveHandler hrManager = new HRManager();

        // 构建责任链
        supervisor.setNextHandler(departmentManager);
        departmentManager.setNextHandler(hrManager);

        // 发起请假请求
        int leaveDays = 4;
        supervisor.handleRequest(leaveDays); // Output: Department Manager approved 4 days leave.

        leaveDays = 8;
        supervisor.handleRequest(leaveDays); // Output: HR Manager approved 8 days leave.

        leaveDays = 12;
        supervisor.handleRequest(leaveDays); // Output: Leave request for 12 days was denied.
    }
}

在这里面，我们可以根据需求新增、删除和编排每个节点。

责任链模式的具体使用场景

  • 审批流程：如请假、报销等多级审批流程。
  • 请求处理链：如在Web应用中，多层过滤器或拦截器的处理链。
  • 错误处理：系统中的错误或异常处理链。
  • 命令处理：将一系列命令依次传递给多个处理者，直到找到能处理该命令的对象。

责任链模式的改进

责任链模式可以与其他模式结合使用，以提高灵活性。例如：

  • 与装饰者模式结合：增强责任链中每个处理者的功能。
  • 与策略模式结合：动态决定链的顺序或使用的处理者。

总结

责任链模式通过将请求的发送者与接收者解耦，使得多个对象都有机会处理请求。它适用于需要灵活地组合处理者来处理请求的场景，能够提高系统的灵活性和可扩展性。然而，责任链模式的使用也需要谨慎，特别是在链较长或请求处理流程复杂的情况下，可能会增加调试的难度。

重温一下责任链模式的关键点：

  • 处理对象的动态组合：责任链模式将处理对象以链式结构组合，使得请求可以沿着这条链依次传递，直到有对象处理它。
  • 请求的传递：请求从链的开始传递到链尾，链中的每个对象都可以选择处理该请求或将请求传递给下一个处理对象。
  • 降低耦合：客户端无须知道请求是由哪一个对象处理的，从而降低了对象之间的耦合度。
