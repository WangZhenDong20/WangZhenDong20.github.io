Linux常用命令及常见问题

Posted by SleepingFace on Monday, August 19, 2024

Linux常用命令及常见问题

前言： 基本上每次都会查一遍命令，就在这里做个简单的记录吧。

Linux常用命令

1. ls

用途： 列出目录内容 使用场景： 查看当前目录下的文件和文件夹。

ls

2. cd

用途： 切换目录 使用场景： 导航到不同的目录。

cd /path/to/directory

3. pwd

用途： 显示当前工作目录 使用场景： 确定当前所在的目录路径。

pwd

4. cp

用途： 复制文件或目录 使用场景： 复制文件或目录到另一个位置。

cp file.txt /path/to/destination/
cp -r directory/ /path/to/destination/  # 复制整个目录

5. mv

用途： 移动或重命名文件/目录 使用场景： 移动文件到另一个位置或重命名文件。

mv file.txt /path/to/destination/
mv oldname.txt newname.txt  # 重命名文件

6. rm

用途： 删除文件或目录 使用场景： 删除不再需要的文件或目录。

rm file.txt
rm -r directory/  # 删除整个目录

7. mkdir

用途： 创建新目录 使用场景： 在指定路径下创建一个新目录。

mkdir new_directory

8. rmdir

用途： 删除空目录 使用场景： 删除一个空的目录。

rmdir empty_directory

9. touch

用途： 创建空文件或更新文件的时间戳 使用场景： 创建一个新的空文件或更新现有文件的修改时间。

touch newfile.txt

10. cat

用途： 显示文件内容 使用场景： 在终端上快速查看文件的内容。

cat file.txt

11. echo

用途： 输出字符串到终端或文件 使用场景： 显示一段文本，或者将文本写入文件。

 echo "Hello, World!"
echo "Hello, World!" > file.txt  # 将文本写入文件

12. grep

用途： 搜索文本中的模式匹配行 使用场景： 在文件中查找包含特定关键字的行。

grep "search_term" file.txt

13. find

用途： 在目录树中查找文件或目录 使用场景： 查找符合指定条件的文件或目录。

find /path/to/search -name "*.txt"

14. chmod

用途： 更改文件或目录的权限 使用场景： 修改文件或目录的读写执行权限。

chmod 755 script.sh  # 使脚本可执行

15. chown

用途： 更改文件或目录的所有者 使用场景： 更改文件或目录的所有者或用户组。

chown user:group file.txt

16. df

用途： 显示文件系统的磁盘使用情况 使用场景： 检查系统中各个分区的磁盘使用情况。

df -h

17. du

用途： 显示文件或目录的磁盘使用情况 使用场景： 检查目录或文件占用的磁盘空间。

du -sh /path/to/directory

18. ps

用途： 显示当前运行的进程 使用场景： 查看系统中正在运行的进程列表。

ps aux

19. top

用途： 实时显示系统的资源使用情况 使用场景： 监控系统资源使用情况，如CPU和内存。

top

20. kill

用途： 终止进程 使用场景： 终止挂起或占用资源的进程。

kill PID
kill -9 PID  # 强制终止进程

21. sudo

用途： 以超级用户权限执行命令 使用场景： 执行需要管理员权限的操作。

sudo apt update

22. apt-get

用途： 包管理工具，用于安装、更新和删除软件包 使用场景： 在Ubuntu系统上安装或更新软件包。

sudo apt-get install package_name
sudo apt-get update
sudo apt-get upgrade

23. wget

用途： 从网络上下载文件 使用场景： 下载远程服务器上的文件到本地。

wget http://example.com/file.zip

24. curl

用途： 传输数据（常用于HTTP请求） 使用场景： 发送HTTP请求或下载文件。

curl http://example.com

25. tar

用途： 压缩和解压文件 使用场景： 创建或解压缩tar归档文件。

tar -czvf archive.tar.gz /path/to/directory  # 压缩
tar -xzvf archive.tar.gz  # 解压缩

26. zip/unzip

用途： 压缩和解压zip文件 使用场景： 创建或解压缩zip归档文件。

zip -r archive.zip /path/to/directory  # 压缩
unzip archive.zip  # 解压缩

27. free

**用途：**显示系统内存的使用情况

**使用场景：**查看系统内存的使用情况。

free -h

常见问题

1. CPU 使用率飙升

异常情况： 系统变得非常缓慢，top 显示 CPU 使用率接近 100%。

排查方法：

  • 使用 top 或 htop 查看高 CPU 占用的进程:

    top
    • 在 top 中按 P 键按 CPU 使用率排序，找到占用 CPU 资源最多的进程。

  • 使用 ps 命令按 CPU 使用率排序:

    ps aux --sort=-%cpu | head -n 10
    • 列出占用 CPU 最多的前 10 个进程。

  • 检查系统日志:

    sudo tail -f /var/log/syslog
    • 查看是否有进程崩溃或其他异常。

解决方法： 如果某个进程占用过多 CPU，可以选择结束该进程（kill PID），或进一步调试该进程以了解其问题。

2. 内存不足

异常情况： 系统的响应速度变慢，程序频繁崩溃，可能出现 Out of Memory 错误。

排查方法：

  • 使用 free -h 检查内存使用情况:

    free -h
    • 查看内存和交换空间的使用情况，判断是否内存不足。

  • 使用 top 或 htop 检查内存使用较高的进程:

    top
    • 按 M 键按内存使用排序，找到占用内存最多的进程。

  • 使用 ps 命令按内存使用排序:

    ps aux --sort=-%mem | head -n 10
    • 列出占用内存最多的前 10 个进程。

解决方法： 尝试关闭或重启占用大量内存的进程，或者增加系统的物理内存或交换空间。

3. 磁盘空间不足

异常情况： 磁盘已满，无法写入新的数据，系统日志显示 No space left on device。

排查方法：

  • 使用 df -h 查看磁盘使用情况:

    df -h
    • 查看各个分区的使用情况，判断哪些分区已满。

  • 使用 du 找出占用大量空间的目录:

    du -sh /* 2>/dev/null | sort -h | tail -n 10
    • 找到占用磁盘空间最多的目录。

  • 查找并删除大文件:

    find /path/to/search -type f -size +100M
    • 查找并删除无用的大文件。

解决方法： 清理不必要的文件或日志，移动或删除不再需要的数据文件，扩展磁盘分区或添加新磁盘。

4. 磁盘 I/O 高

异常情况： 系统响应速度慢，磁盘操作（如读取、写入）非常缓慢。

排查方法：

  • 使用 iostat 查看磁盘 I/O 使用情况:

    sudo apt-get install sysstat  # 安装 iostat
iostat -x 1 10
    • 查看各个设备的 I/O 情况，确定是否有磁盘瓶颈。

  • 使用 iotop 查看具体进程的 I/O 使用情况:

    sudo apt-get install iotop
sudo iotop
    • 找到占用大量 I/O 的进程。

解决方法： 如果某个进程占用过多的磁盘 I/O，可以考虑优化该进程，或者将 I/O 密集型任务分配到其他磁盘。

5. 网络连接缓慢或不稳定

异常情况： 网络速度慢，丢包率高，连接频繁中断。

排查方法：

  • 使用 ping 测试网络连通性:

    ping google.com
    • 检查与目标主机的连通性和响应时间，判断是否存在网络延迟或丢包。

  • 使用 ifconfig 或 ip a 查看网络接口状态:

    ifconfig
    • 检查网络接口是否正常工作，是否有丢包或错误。

  • 使用 traceroute 跟踪路由路径:

    traceroute google.com
    • 查看数据包到达目标的路由路径，确定是否存在网络瓶颈。

  • 使用 netstat 查看网络连接状态:

    netstat -tuln
    • 查看系统的网络连接，检查是否有异常连接或端口占用。

解决方法： 如果发现网络异常，可以联系网络管理员，检查网络设备或重新配置网络接口。
