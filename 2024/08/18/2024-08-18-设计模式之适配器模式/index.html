设计模式之适配器模式

Posted by SleepingFace on Sunday, August 18, 2024

适配器模式（Adapter Pattern）

概述：

适配器模式是一种结构型设计模式，它允许你将一个类的接口转换成客户端所期待的另一个接口，使得原本由于接口不兼容而不能一起工作的类可以一起工作。

关键点：

  • 接口兼容：适配器模式使得两个不兼容的接口可以一起工作，通常是为了让一个类可以与一个接口不兼容的类进行合作。
  • 目标接口与被适配者：适配器提供一个目标接口，并将请求委派给被适配者，以便客户端可以通过适配器使用被适配者的功能。
  • 两种实现方式：对象适配器（基于组合）和类适配器（基于继承）。

具体场景：

假设我们有一个旧系统，它使用一个早期开发的类LegacyRectangle来表示矩形。这个类只有draw(x1, y1, x2, y2)方法，通过两个对角点的坐标绘制矩形。而新系统中，矩形是通过Rectangle类来表示的，提供了draw(width, height)方法。现在，你需要在新系统中复用旧系统中的LegacyRectangle类。

结构：

适配器模式包含以下几个主要角色：

  • 目标接口（Target）：定义客户需要的接口。
  • 适配者类（Adaptee）：定义一个已经存在的接口，这个接口需要适配。
  • 适配器类（Adapter）：实现目标接口，并通过组合或继承的方式调用适配者类中的方法，从而实现目标接口。

示例代码：

1. 旧系统中的LegacyRectangle类

// 被适配者类：旧系统中的矩形类
public class LegacyRectangle {
    public void draw(int x1, int y1, int x2, int y2) {
        System.out.println("Drawing rectangle with corners at (" + x1 + "," + y1 + ") and (" + x2 + "," + y2 + ")");
    }
}

2.新系统中的Rectangle接口

// 目标接口：新系统中使用的矩形接口
public interface Rectangle {
    void draw(int width, int height);
}

3.适配器类

// 适配器类：将LegacyRectangle适配为Rectangle
class RectangleAdapter implements Rectangle {
    private LegacyRectangle legacyRectangle;

    public RectangleAdapter(LegacyRectangle legacyRectangle) {
        this.legacyRectangle = legacyRectangle;
    }

    @Override
    public void draw(int width, int height) {
        // 将宽高转换为对角点坐标，并调用LegacyRectangle的方法
        int x1 = 0;
        int y1 = 0;
        int x2 = width;
        int y2 = height;
        legacyRectangle.draw(x1, y1, x2, y2);
    }
}

4.Client代码

public class AdapterPatternDemo {
    public static void main(String[] args) {
        // 旧系统中的LegacyRectangle类实例
        LegacyRectangle legacyRectangle = new LegacyRectangle();

        // 使用适配器将旧系统的类适配为新系统中的接口
        Rectangle rectangle = new RectangleAdapter(legacyRectangle);

        // 使用新系统的接口绘制矩形
        rectangle.draw(10, 20);
    }
}

使用场景：

  • 遗留代码复用：当你想要复用一个已有类，而这个类的接口不符合当前的系统要求时，可以使用适配器模式将其接口转换为系统期望的接口。

  • 接口不兼容的类之间协作：适配器模式可以在接口不兼容的类之间架起桥梁，使它们能够协同工作。

  • 第三方库集成：当引入一个第三方库，库中的类接口与系统期望的接口不一致时，可以通过适配器模式进行集成。

优缺点

优点：

  • 提高类的复用性：通过适配器模式，可以轻松地复用已有的类，而不需要对类本身进行修改。
  • 增强类的灵活性：适配器模式允许你在不改变已有代码的情况下，兼容新的接口或功能需求。
  • 符合开闭原则：你可以通过引入适配器来扩展系统的功能，而不需要修改现有的代码。

缺点：

  • 增加复杂度：引入适配器层可能会增加系统的复杂性，特别是在有多个适配器链的情况下。
  • 性能开销：由于适配器引入了额外的调用，可能会有轻微的性能开销，尤其是在需要高性能的系统中。

适配器模式的变体

  1. 双向适配器（Two-way Adapter）：一个适配器能够同时适配两个方向，即它既可以适配目标接口到被适配者，也可以反过来适配。
  2. 默认适配器（Default Adapter）：当接口中有很多方法时，可以使用默认适配器模式。默认适配器实现接口中的所有方法，子类可以只重写其中感兴趣的方法。

总结

适配器模式用在需要将现有代码集成到新系统中或让接口不兼容的类协作时。它通过引入一个适配器层，解除了客户端和具体实现之间的耦合，增强了系统的灵活性和可扩展性。适配器模式的应用场景广泛，特别适合在遗留系统维护、第三方库集成以及接口不兼容的类之间的协作中使用。

重温一下适配器模式的关键点：

  • 接口兼容：适配器模式使得两个不兼容的接口可以一起工作，通常是为了让一个类可以与一个接口不兼容的类进行合作。
  • 目标接口与被适配者：适配器提供一个目标接口，并将请求委派给被适配者，以便客户端可以通过适配器使用被适配者的功能。
  • 两种实现方式：对象适配器（基于组合）和类适配器（基于继承）。
